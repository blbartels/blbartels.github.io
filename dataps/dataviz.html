4  Data Visualization

The topic of data visualization has become one the more exciting developments in data analysis in the contemporary era. Scholars and analysts have thought systematically about ways to distill complex information (in the form of large amounts of data) into graphical displays that people can readily understand and interpret. How many times have you heard it? “I consider myself a visual learner.” Many people fit this bill, of course. Presenting results visually maximizes a researcher’s ability to communicate their findings with clarity.

Data visualization developments have greatly benefited from the R package called ggplot2, which is perhaps the crown jewel of the tidyverse. This chapter will simultaneously introduce readers to the foundations and developments of data visualization and the basic and advanced features of ggplot. The chapter will first cover some basic data visualization principles that apply to all graphical displays. Readers interested in additional information about these principles can consult Wickham, Cetinkaya-Rundel, and Grolemund (2023). Next, the chapter covers the nuts and bolts of univariate graphical displays, i.e., summaries of just one variable where we want to evaluate and communicate the central tendency and dispersion of a variable. The third topic covers the nuts and bolts of bivariate graphical displays, i.e., summaries of the strength of an association or relationship between two variables.

This chapter will also emphasize the importance of a variable’s level of measurement — nominal, ordinal, or interval/continuous from Section 2.2.3 — as the critical driver of what type of graph a user will generate to communicate univariate and bivariate descriptions.

4.1 Data Visualization Principles

When users present a graphical display of some aspect of their data, I recommend that they “sweat the small stuff.” After all, the researcher is distilling sometimes vast amounts of information into a single plot, so one should take great care that the graph accurately conveys the intended summary information. Let’s begin with a very simple graph of a normal distribution, which is a univariate display that we will cover in the next section in greater detail. Most users are familiar with the well-known normal distribution as a bell-shaped, symmetric distribution that is centered around a single central tendency. The code below simulates a “random variable” for 100,000 observations based on the normal distribution.

library(tidyverse)

set.seed(123)
mynorm <- data.frame(id=1:100000) |> 
  mutate(score=rnorm(n = 100000, mean = 0, sd = 1))

Note that I first “set a seed,” which allows me to replicate exactly the data that I have simulated (in this case the “score” variable) in future instances. If I set the same seed in the future, I will get the same underlying data values for score. The command creates a dataset object, “mynorm,” with 100,000 observations. I then create the score variable in this dataset which is a random variable based on a normal distribution via the use of the rnorm command. I next set the values of the mean and standard deviation (which the next section will discuss in greater detail). A distribution with a mean of 0 and a standard deviation of 1 is called a “standard normal distribution.” Each unit of measurement (from 1 to 2 to 3) for the score variable represents one standard deviation. Next, I use ggplot to create a “density plot.”

mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density() + 
  xlim(-4, 4)

Since ggplot is part of the tidyverse, we can use piping, which I use here when I first call up the dataset “mynorm.” Inside the ggplot command is aes(), which stands for “aesthetic.” The aesthetic of a ggplot graph includes anything involving the basic structure of the graph, that is, what is included within the actual plot itself. Within the aesthetic is where I declare what variable will go on the x-axis. In this case, that is the “score” variable, which means I enter “x=score.” The full command for setting the aesthetic is actually mapping = aes(x=score). However, ggplot understands the shortcut aes(x=score) as identical. Moreover, without the use of piping, we would type ggplot(data=mynorm, aes(x=score)) or for an identical shortcut, we could remove data= like this, ggplot(mynorm, aes(x=score)). Because piping becomes ever so useful in future contexts, I will use piping even in these more straightforward contexts.

Note that within the aesthetic for this graph, we do not specify a y variable for the y-axis. That is because a density plot is a frequency distribution (also a probability distribution). As I will discuss below in Section 4.2, the y-axis in this plot represents the frequency of observations at a particular x-value. The higher the plot on the y-axis, the more observations there are at that point. A density, which is a quantity central to the normal distribution, is proportional to a probability and a frequency.

The following line in the ggplot command declares what “geometry” I will use. Since I want a density plot, I use geom_density. I will give a full listing of univariate geometries below. Finally, in the third line, I tell ggplot what the range of the x-axis should be, ranging from -4 to 4. If I do not issue this command, ggplot will use defaults, which are often sensible and are good to use at the start. Importantly, note that each specification is separated by a + sign. My own practice is hit “enter” after each plus and make each specification a separate line. Note that even though this ggplot command has four lines, it is one command.

I will now use this basic plot to walk through some basic principles of data visualization. When generating a graphical display, a researcher should make it a “stand-alone” plot to the greatest extent possible. Though a graph is always accompanied by textual interpretation, a researcher should strive to generate a graph that someone could observe and immediately glean a sufficient amount of information in that graph to make a basic interpretation of what it communicates.

4.1.1 Titles and Axis Labels

The first step in achieving this “stand-alone” goal requires giving the graph: (1) a descriptive graph title, (2) clear titles for the x-axis and y-axis, and (3) sensible numeric labels and increments for the x-axis and y-axis. The command below attempts to achieve all three goals using the labs, scale_x_continuous, and scale_y_continuous specifications added on to the ggplot command.

mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density() + 
  labs(title="Density Plot for a Normal Distribution", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-4, 4), breaks = seq(-4, 4, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1))

To include a graph title and titles for the x-axis and y-axis, I can use labs. The syntax is very self-explanatory. Simply separate each title by a comma. Because the score variable is continuous, or an interval variable, I use scale_x_continuous to specify values of the x-axis. In this case, I actually prefer to include x-axis numeric labels in increments of 1. The limits specification defines the minimum and maximum limits of the x-axis (like xlim did in the prior graph). breaks specifies which numeric labels I want to include. This syntax, seq(-4, 4, 1), means make numeric labels go from -4 to 4 in increments of 1.1 If I were to put a “2” at the end instead of a “1” (i.e., seq(-4, 4, 2)), the x-axis would include numeric labels -4, -2, 0, 2, 4. For the y-axis, I like the default provided by ggplot, but I formalize it in the scale_y_continuous command.

So far so good with our objectives for titles and axis labels. One final consideration concerns font sizes for all the facets we have worked with. By default, the title has the largest font, the axis titles have the second-largest font, and the axis numeric labels have the smallest font. Also, some researchers will want to center the graph title. We can change these features in the theme() command in ggplot. Let’s center the graph title and set the font size to be the same (12pt) for all features to see what it looks like.

mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density() + 
  labs(title="Density Plot for a Normal Distribution", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-4, 4), breaks = seq(-4, 4, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1)) +
  theme(plot.title = element_text(hjust = 0.5, size=12), # Graph title
        axis.title = element_text(size = 12),  # Axis titles
        axis.text  = element_text(size = 12))  # Axis tick labels

Within the theme command, I have included options for plot.title (for the overall graph title), axis.title (for the x-axis and y-axis titles), and axis.text (for x-axis and y-axis numeric labels). Typically, we want the elements of the x-axis and y-axis to match. However, options exist for making disctinctions between the axes. For each of these three specifications within theme, we specify settings for the element_text. For plot title, I use hjust=0.5 to center the graph title. I use size=12 for all three features to make the font size 12pt for all text in the graph. Some researchers might actually prefer a hierarchy of font sizes, as the default graph above did. Let’s use a 10pt font the axis numeric labels and 12pt font for the titles:

mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density() + 
  labs(title="Density Plot for a Normal Distribution", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-4, 4), breaks = seq(-4, 4, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1)) +
  theme(plot.title = element_text(hjust = 0.5, size=12), # Graph title
        axis.title = element_text(size = 12),  # Axis titles
        axis.text  = element_text(size = 10))  # Axis tick labels

Users will naturally experiment with different font sizes. One wants make sure to accommodate accessibility concerns by making the font big enough and readable enough. Font sizes smaller than 10pt can get hard to read.

Finally, the second row of the figure above changes the line style to “dashed”. Other options for line style (which can be used in other graphs with a line in them) include: “dotted”, “dotdash”, “longdash”, and “twodash.” The last graph uses the fill option with “dodgerblue”, which is often appealing for a graph like this because it clearly delineates the area under the curve relative to the other space. 4.1.3 General Look and Themes

So far, all graphs have included a gray background with gridlines (for both axes) included by default. We can change the general “look” of a graph adding theme_[type]() as a separate line in the graph. ggplot has several themes. If you are using ggplot, once you begin typing “theme_…”, RStudio will give you a pulldown menu of all the options. Later, I compare four commone themes. First, to show the general syntax, I change the theme below to theme_minimal().

mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density(fill="dodgerblue", alpha=.4, linewidth=.2) + 
  labs(title="theme_minimal()", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-4, 4), breaks = seq(-4, 4, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1)) +
  theme_minimal() +
  theme(plot.title = element_text(hjust = 0.5, size=12), 
        axis.title = element_text(size = 12),  
        axis.text  = element_text(size = 10))

Always make sure to put theme_minimal() (or whatever theme you specify) before theme() since the latter tweaks aspects of the former. This theme is quite bare bones but it does include core elements, like gridlines in the background. Below, I compare the default and minimal to two other general themes, theme_bw() and theme_classic().

In political science research, I have found that the default, minimal, and bw tend to be the most popular. I have also included classic, with its x-axis and y-axis only and also no gridlines. With the specific use of the x-axis and y-axis, a literal data visualization principle would suggest that classic should only be used if the intersection of the x and y-axes is truly the origin (i.e., 0 for both x and y). That fits the bill for the y-axis in this case but not for the x-axis.

Another data visualization consideration is the “padding” that ggplot adds by default on both ends of each axis. The horizontal padding at the bottom of the graphs is most glaring in this graph because the “floor” is supposed to be zero (which is the minimum possible value of y in this case). Note however the visible gap below this floor in the default, bw, and classic graphs. Because of the lack of a background (as in the default) or frame (as in bw), the padding issue is not glaring in the minimal theme.

To take out the gap on the bottom in theme_bw(), for instance, we could use the expand option within scale_y_continuous. ggplot adds a 5% padding gap on both sides of each axis. That is satisfactory for the x-axis because it ranges from -4 to 4, and we want the padding to exist on the ends to suggest that these are not necessarily theoretical limits. Focusing on the y-axis, however, we want to indicate that 0 is a limit (the minimum, or floor) yet 0.4 is not necessarily a limit. Thus, within scale_y_continuous, I use the expand option to keep the 5% padding at the top (the 0.05, which means 5%, within mult = c(0.003, 0.05)) and I reduce the padding at the bottom to (near) 0 (the 0.003). Why not just use 0 instead of 0.003? I have found that when I use 0, the 0 tick appears just below the bottom border, whereas I like to align the 0 tick with the bottom border.

mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density(fill="dodgerblue", alpha=.4, linewidth=.2) + 
  labs(title="theme_bw(), removing the bottom padding", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-4, 4), breaks = seq(-4, 4, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1), 
                     expand = expansion(mult = c(0.003, 0.05))) +
  theme_bw() +
  theme(plot.title = element_text(hjust = 0.5, size=12, face = "bold"), 
        axis.title = element_text(size = 12),  
        axis.text  = element_text(size = 10))

Note the difference between this graph and the prior graph that used theme_bw() — it now reduces the padding gap at the bottom.

The panel.grid options in theme allow for alterations to the gridlines. The first graph eliminates all gridlines, while the second one includes “major gridlines” only (by suppressing panel.grid.minor). Note that ggplot specifies “major” versus “minor” gridlines. The major gridlines are those corresponding to the numeric labels on the axes. The minor gridlines are those halfway in between the major gridlines. On the bottom row, I report major gridlines only but change the linestyle to “dotted.” The lower-left graph uses the default color with dotted lines, which are barely visible; note that “dashed” lines using the default color are more visible. The last graph darkens the dashed gridlines to “gray70,” which makes the lines more visible.

With some basic data visualization principles in hand, we can now proceed to generating and interpreting univariate graphs.

4.2 Univariate Graphical Displays

Univariate graphical displays provide a visual means of describing a variable’s central tendency and dispersion, or distribution. We will eventually link these visual descriptions to more formal statistical descriptions in Chapter 6. Researchers want to summarize variables of interest both as a way of understanding those variables generally and also for reasons connected to appropriate ways to examine variables with formal statistical procedures. Thus, data visualization facilitates appropriate statistical analysis. As I will discuss in greater detail, univariate graphs are also directly connected to levels of measurement (à la Section 2.2.3) both in terms of the foundations for what type of graph to use and the implications for types of information we want to report and statistical analyses we want to conduct.

4.2.1 Dispersion

Dispersion describes how spread out or close together observations are on values of the variable. Many researchers will also describe dispersion relative to the central tendency, or the average value. Are many of the observations tightly packed around the average value or are they widely dispersed around the mean? This diserpsion can also communicate how informative the central tendency is as a generalization. The more dispersion, the less informative a central tendency’s generalization is. The lower the dispersion, the more informative the generalization is.

The figure below illustrates this basic type of variation in dispersion.

set.seed(123)
mynorm2 <- data.frame(id=1:100000) |> 
  mutate(score=rnorm(n = 100000, mean = 0, sd = 2))

a <- mynorm |> 
  ggplot(aes(x=score)) + 
  geom_density(fill="dodgerblue", alpha=.4, linewidth=.2) + 
  labs(title="Low Dispersion", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-6, 6), breaks = seq(-6, 6, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1), 
                     expand = expansion(mult = c(0.003, 0.05))) +
  theme_bw() +
  theme(plot.title = element_text(hjust = 0.5, size=12, face = "bold"), 
        axis.title = element_text(size = 12),  
        axis.text  = element_text(size = 10)) 

b <- mynorm2 |> 
  ggplot(aes(x=score)) + 
  geom_density(fill="dodgerblue", alpha=.4, linewidth=.2) + 
  labs(title="High Dispersion", 
       x="Score (Random Variable)", y="Density") +
  scale_x_continuous(limits = c(-6, 6), breaks = seq(-6, 6, 1)) +
  scale_y_continuous(limits = c(0, .4), breaks = seq(0, .4, .1), 
                     expand = expansion(mult = c(0.003, 0.05))) +
  theme_bw() +
  theme(plot.title = element_text(hjust = 0.5, size=12, face = "bold"), 
        axis.title = element_text(size = 12),  
        axis.text  = element_text(size = 10)) 

a + b + plot_layout(ncol = 2)

First, the left graph is the one that we analyzed earlier (“mynorm”). The right graph is based a second random number I simulated using the code above. Instead of a standard deviation of 1, I specified a standard deviation of 2. Thus, this variable is a random variable derived from a normal distribution with mean=0 and standard deviation=2. By definition, we know that this second variable will have a higher degree of dispersion. From the “rules of standard deviation” (that we will discuss in Chapter 6), we know that 95% of all observations are within about 2 standard deviations from the mean. In the left graph, that means 95% of observations are \(\pm\) 2 units from the mean (between -2 and 2, since the mean = 0). In the right graph, 95% of the observations are \(\pm\) 4 units from the mean (between -4 and 4).

Comparing the graphs brings these comparisons to “visual life.” The left graph is a “high and tight” distribution, where observations are packed around the mean and many observations are at or very close to the mean value (of 0). The right graph is “lower and flatter,” where observations are more spread out around a wider range of the “score” variable. Note that much fewer observations at or near the mean vis-à-vis the low height of the curve at and around the mean. The high and tight, low-dispersion left grahp also shows that the mean is more informative generalization of the “score” variable than the low and flat, high-dispersion graph on the right.

Another way to talk about these variations in dispersion is in terms of “homogeneity” and “hetergeneity.” The left graph shows a much more homogeneous set of observations on this characteristic (“score”); many people tend to resemble each other. The right graph shows more heterogeneity; people are generally quite different from each other on this characteristic.

Though these graphs differ in the amount of dispersion, they share a key feature of what we want to visualize in univariate graphs: Whether or not the variable’s distribution is symmetric, bell-shaped, and “single-peaked.” In many ways, univariate data visualization centers on ascertaining whether our univariate distribution deviates from such a symmetric, bell-shaped, and single peaked distribution like those above. The graphs below show three such common deviations compared to the “standard” bell-shaped distribution.

Graph A provides a “benchmark” for comparison — the distribution is bell-shaped, symmetric, and “single-peaked,” or unimodal. This last aspect means that there’s just one central tendency. By benchmark, I do not mean that this is an “ideal” or desirable distribution to have for your own variables. In fact, variables rarely resemble exactly such a well-behaved distribution. Data can be messy. However, the properties I have discussed surrounding this distribution provide key points of comparison and discussion when describing a variable.

Graph B shows one important deviation from the normal distribution in graph A: It is a bimodal distribution. Instead of having just one central tendency, it has two. In politics, for instance, we are often interested in the distribution of voters’ policy preferences across policy issues like immigration, gun control, economics, abortion rights, and the environment. Graph A depicts a situation where there is public consensus around a more moderate policy position. It suggests a consensus could emerge between Democrats and Republicans on a compromise.

Graph B, however, shows the absence of consensus: It depicts polarization on an underlying issue, with two groups — Democrats and Republicans — advancing very different policy preferences on an issue. Very few people take a middle position, meaning compromise is unlikely. In American politics, we often say that this is recipe for “gridlock” in Congress or other settings. Beyond this example, a bimodal distribution suggests the presence of distinct groupings of individuals. We may see this in approval ratings or candidate evaluations, where there may be one group that is highly approving of a candidate or policy and another that is disapproving. Some will intepret such graphs as saying the public is of “two minds.”

Graphs C and D display “skewed” variables. Skewness is a clear deviation from graph A’s “symmetry” property. Graph C shows that the mass of observations is tilted to the lower end of X, and there small amounts of observations at very high values of X. These “outliers” at the high end (the “extremes”) are what generate the skewness. Graph C is thus called a “positively skewed” variable, or “skewed to the right.” This latter description may initially confuse readers — after all, the mass of observations is shifted to the “left.” But the “skewed to the right” is describing the direction of the skewness as in the above description of how the more high-value outliers on the “right” side (or high end of X) are generating the skewness. Income or GDP measures are classic positively skewed variables. Imagine Graph C describes something like household income in the United States. The mass of individuals will be packed around some more typical value but there will be many high-income and then ultra-wealthy individuals at the very high ends. These outliers pull that right tail out to high ends. There are very few people at that high end, but we cannot just cut them off.

Graph D presents the opposite direction of skewness: Negative skew, or skewed to the left. Here, the mass of observations are tilted to the right, with the more extreme outlier observations skewing the distribution to the left, on the low end X. An example of a negatively skewed variable is human lifespan, or age of death, around the world. In today’s age, life expectancy represents the typical age (central tendency) people will live to. If worldwide life expectancy is around 75 years of age, there are lots of observations that will be packed around that central tendency. However, we know that some people defy life expectancy and die young due to health problems or untimely accidents. Those individuals low on X are the outliers that skew the variable.

The ultimate value of data visualization is the ability to diagnose the nature of a variable’s distribution before one begins doing statistical analysis on that variable or analysis in conjunction with another variable. The nature of a variable’s dispersion and distribution essentially dictates how we interpret central tendency (as Chapter 6 will discuss in greater detail). Moreover, for some variables (like in graphs B-D), it makes more sense to describe the variable in terms of dispersion and distribution first before describing its central tendency. If one has a variable with a bimodal distribution, one should describe it that way first. That’s the key characteristic of the variable that one wants to communicate. Data visualization gives us that interpretation. Same with skewness: The first thing we want to say about graph C is that it is positively skewed.

These descriptions set up a critical step in describing a skewed variable’s central tendency and dispersion, where one wants to think of skewed variables in terms of percentiles: The median, or 50th percentile, as well as the 25th and 75th percentiles for dispersion. These percentiles are more “immune” from the skewness induced by extreme values and outliers, whereas skewness directly affects statistics like the mean and standard deviation. Once again, Chapter 6 will cover these topics in greater depth.

4.2.2 Univariate Graphs and Levels of Measurement

Let’s now dive into types of univariate graphs used to describe variables. Importantly, the selection of the graph to use is directly connected to levels of measurement. For example, the density plots from above are clearly most appropriate for interval/continuous variables. It would make zero sense to do a density plot for a nominal (unordered) variable with 4 or 5 discrete categories. It may make sense, under certain circumstances, to do density plots for ordinal variables with several categories — these variables can sometimes be in a gray area.

4.2.3 Density Plots

Let’s continue discussion of density plots with an example discussed briefly above: polarization. We will look at the distribution of the U.S. House of Representatives to see if the members’ “policy preferences” (their generalized ideological views on policy issues ranging from liberal to conservative) do indeed follow a bimodal distribution, as the polarization perspective would suggest. The Voteview website houses the widely-used “Poole and Rosenthal NOMINATE scores.” These scores are named after political scientists Keith Poole and Howard Rosenthal, who developed a scaling procedure and measurement model to generate estimates of ideology/policy preferences for every member of Congress (as well as every president) in history (see, e.g., Poole and Rosenthal 1997). The scores range from about -1 to 1, with increasing scores reflecting greater conservatism. One can generally think of values around 0 as more moderate, negative values as more liberal, and postive values as more conservative.

The dataset for all members throughout history is available (as a .csv file) at the Voteview website. Below, I read in that dataset directly from this website (using the same procedures from Chapter 3) and filter it to contain only members of the House of Representatives for the 119th Congress, which commenced in January of 2025.2 The dataset contains 451 members, some of whom are non-voting members. I remove those observations using the “district_code” filter.3 The dataset includes multiple members per congressional district for situations where one member dies or retires and another replaces him or her sometime during the Congress. The group_by command gets down the district level, and the slice_max operates similarly to the “cluster count” and “cluster size” variables we worked with in Section 3.4.3. The slice_max command tells RStudio to keep just 1 observation per congressional district (n = 1) based on which member cast the most votes (and slice_min would select the observation with the fewest votes).

house <- 
  read_csv("https://voteview.com/static/data/out/members/HSall_members.csv") |> 
  filter(chamber=="House", district_code > 0) |> 
  filter(congress==119) |> 
  group_by(state_abbrev, district_code)  |> 
  slice_max(nominate_number_of_votes, n = 1)  |> 
  ungroup()

house |> 
  ggplot(aes(x = nominate_dim1)) +
  geom_density() +
  labs(title="Polarization in the\nHouse of Representatives, 2025", 
       x="Ideology Score (NOMINATE), Ranging from Lib. to Cons.", y="Density") + 
  scale_x_continuous(limits = c(-1, 1), breaks = seq(-1, 1, .5)) +
  theme_minimal() +
  theme(plot.title = element_text(hjust = 0.5, size=12, face = "bold"),
        axis.title = element_text(size=10), 
        axis.text  = element_text(size = 10))

The figure shows the distribution of preferences, using a density plot. The plots displays a clear, bimodal distribution of preferences, which indicate polarization. This graph could be sufficient for explaining polarization in the House. We essentially infer that the mass on the left is Democrats and the mass on the right is Republicans. Or we could also make that more explicit by adding color, essentially delineating which observations are Democrats and which are Republicans.

The ggplot command below does just that. I will explain the syntax first. The dataset includes a party variable called “party_code”; Democrats=100, Republicans=200. The 119th House contains exclusively Democrats and Republicans. Within the aesthetic, I have included fill=factor(party_code)). First, like past density plots in this chapter, the fill command will insert color within the area under the density curve. Here, though, I am telling ggplot to based that fill on whether members are Republicans or Democrat, which is the party_code variable. Back to levels of measurement and their relation to RStudio syntax, party_code is a “numeric” variable (do class(house$party_code) on your own). But to use the fill variable, I want ggplot to treat party as a “discrete” variable (essentially a nominal variable); that is, there are two groups, one will be one color and the other will be another color. Thus, in the fill command, I tell ggplot to treat party_code as a “factor” variable instead of a numeric variable. In RStudio, a factor variable is a nominal variable. That is exactly what we want in this situation.

Then, in the next line, I tell ggplot exactly how I want to delineate Republicans versus Democrats. I use scale_fill_manual, which is going to be directly connected to fill from the aesthetic, to specify that Democrats should be “dodgerblue” and Republicans should be “red.” I specify the verbal labels for each that will appear in the legend. Finally, name=NULL suppresses a title for the legend; in this case, it is evident what the legend is doing, so it is sufficient to include just Democrats and Republicans. Note the use of c(), which means a column vector. These are the values associated with the factor variable one specifies in the fill command within the aesthetic. How do I know which order to put Republicans or Democrats, blue or red? The answer is that it should go in the same order as the “party_code” variable, where 100=Democrats and 200=Republicans. Since 100 is lower than 200, Democrats should go first in the column vector, and Republicans should go second. ggplot automatically associates the first entry in the column vector with the lowest value for the fill variable and the second entry with next highest value, and so on.

house |> 
  ggplot(aes(x=nominate_dim1, fill=factor(party_code))) + 
  geom_density(alpha=.6, linewidth=.2) +
  scale_fill_manual(values = c("dodgerblue", "red"),
                    labels = c("Democrats", "Republicans"),
                    name=NULL) +
  labs(title="Polarization in the\nHouse of Representatives, 2025", 
       x="Ideology Score (NOMINATE), Ranging from Lib. to Cons.", y="Density") + 
  scale_x_continuous(limits = c(-1, 1), breaks = seq(-1, 1, .5)) +
  theme_minimal() +
  theme(legend.position = "bottom", 
        plot.title = element_text(hjust = 0.5, size=12, face = "bold"),
        axis.title = element_text(size=10), 
        axis.text  = element_text(size = 10))

This figure now provides a quite compelling visualization of the distribution of preferences. It basically displays the definition of party polarization in Congress. Polarization is defined as: (1) within-group homogeneity and (2) between-group distance. The figure displays both features. Notice that there is no overlap in preferences between Democrats and Repblicans. There are very few members in the middle of the ideology distribution, i.e., very few moderates. The central tendencies for Democrats and Republicans are liberal and conservative and quite far apart from each other. Homogeneity among the Democrats is higher than it is for Republicans. In sum, this polarization example is a classic example of a bimodal distribution in political science, and we have connected bimodaility to a substantive insight about American politics today.

A way to extend density plots and their connection to substantive insights is to compare plots across multiple Congresses to visualize change over time. One way to do this expeditiously is by using a “facet wrap,” which can give us separate density plots for separate Congresses. In the data command below, I once again use the full dataset from Voteview, but in the second filter command, I tell RStudio to extract the 79th to 119th Congresses in increments of 5; I select every 5th Congress (79th, 84th, etc.). I also create a “year” variable that is the starting year for each Congress. Since the first Congress was in 1789, I can use that last mutate command to associate years with each Congress.

house2 <- 
  read_csv("https://voteview.com/static/data/out/members/HSall_members.csv") |> 
  filter(chamber=="House", district_code > 0, party_code<300) |> 
  filter(congress %in% seq(79, 119, by = 5)) |> 
  group_by(congress, state_abbrev, district_code)  |> 
  slice_max(nominate_number_of_votes, n = 1)  |> 
  ungroup() |> 
  mutate(year = 1789 + (congress - 1) * 2)

house2 |> 
  ggplot(aes(x = nominate_dim1, fill=factor(party_code))) +
  geom_density(alpha=.6, linewidth=.2) +
  scale_fill_manual(values = c("dodgerblue", "red"), 
                    labels = c("Democrats", "Republicans"), 
                    name=NULL) +
  facet_wrap(~year) +
  labs(title="Partisan Polarization in the House of Representatives Over Time", 
       x="Ideology Score (NOMINATE), Ranging from Lib. to Cons.") +
  theme_minimal() +
  theme(legend.position = "bottom", 
        plot.title = element_text(hjust = 0.5, size=12, face = "bold"),
        axis.title.y = element_blank(), 
        axis.title.x = element_text(size=10), 
        axis.text  = element_text(size = 10))

This graph shows the changing nature of partisan polarization from 1945-2025, every 10 years (which, again, is every 5th Congress). What we want to visualize is changes over time in those two core aspects of polarization: within-party homogeneity in preferences and between-party distance. While this graph does allow for such a visualization, an even better visualization would be to observe these changes in a more vertical fashion — have one Congress per row and just one column. We could do that using facet_wrap (e.g., facet_wrap(~year, ncol=1)), but we would have separate graphs for each with the years above each graph, creating more vertical distance between the graphs. A very powerful data visualization tool is what is called a “ridge plot,” which creates a more compact, vertical comparison. Users should first install the “ggridges” package. The syntax is simple. Everything is basically similar to our prior density plots, except that we need to specify a y variable, which will be “year” in this case because this year variable will go down the vertical side of the plot. We change the geometry to geom_density_ridges, which can only be used if we load the ggridges package.

library(ggridges)

house2 |> 
  ggplot(aes(x = nominate_dim1, y=factor(year), fill=factor(party_code))) +
  geom_density_ridges(alpha=.6, linewidth=.2) +
  scale_fill_manual(values = c("dodgerblue", "red"), 
                    labels = c("Democrats", "Republicans"), 
                    name=NULL) +
  labs(title="Partisan Polarization in the House of Representatives\nOver Time", 
       x="Ideology Score (NOMINATE), Ranging from Lib. to Cons.") +
  theme_minimal() +
  theme(legend.position = "bottom", 
        plot.title = element_text(hjust = 0.5, size=12, face = "bold"),
        axis.title.y = element_blank(), 
        axis.title.x = element_text(size=10), 
        axis.text  = element_text(size = 10))

The ridge plot provides a better visualization for the key aspects we want to compare over time. The ridge plot captures classic descriptions of polarization in Congress over time. We can observe that in general, between-party distance (the distance between the general central tendencies of Democrats and Republicans) has increased over time. Overlap in preferences occurred in the 1940s through the 1980s and even a little in the 1990s. That overlap has disappeared. Also, Democrats have come slightly more homogeneous over time, while Republicans have more heterogeneous over time. For more on polarization in Congress, see, e.g., Rohde (1991); McCarty, Poole, and Rosenthal (2006); and McCarty (2019).
